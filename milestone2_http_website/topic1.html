From Tim Berners-Lee to HTTP/1.1

HTTP started as a minimal protocol (HTTP/0.9) that only supported GET and plain HTML. HTTP/1.0 (mid-1990s) introduced headers and content types. HTTP/1.1 standardized persistent connections, chunked encoding, and caching strategies which improved performance and scalability.

Key milestones

  • 1990 — HTTP/0.9: simple GET requests
  • 1996 — HTTP/1.0: headers and content negotiation
  • 1999 — HTTP/1.1: persistent connections, chunked transfer, more status codes

Why it mattered

These changes allowed web pages to include images, scripts, and stylesheets and laid the groundwork for dynamic web applications.
